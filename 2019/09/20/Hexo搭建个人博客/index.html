发表于 2019-09-20 更新于 2019-09-22

Hexo搭建博客遇到的各种问题及解决（个人笔记）

总结一下我Hexo搭建博客遇到各种各样的问题及解决方法。

git和vim

Win10没有git和vim，在搭建之前需要去官网下载并配置。
Git官方下载 https://git-scm.com/downloads/;
vim官方下载 https://www.vim.org/download.php。

git

注意环境变量配置。

vim

问题：在使用vim是，我发现创建的文章，中文都是乱码。

解决：找到gvim的安装路径，里面有一个这样的文件
使用记事本打开，在开头添加如下：

set nocompatible
set encoding=utf-8
set fileencoding=utf-8,chinese,latin-1
if has(“win64”)
set fileencoding=chinese
else
set fileencoding=utf-8
endif
source $VIMRUNTIME/delmenu.vim
source $VIMRUNTIME/menu.vim
language messages zh_CN.utf-8
set gfn=Consolas:h11
set nu
set cindent
source $VIMRUNTIME/vimrc_example.vim
source $VIMRUNTIME/mswin.vim
behave mswin

然后保存退出就好啦。对了，可能会遇到权限不够，无法保存，这个更改一下权限就OK了。

GitHub

问题：将博客部署到GitHub上时，输入hexo d，出现错误
解决：1. 启动Git Bash，进入命令界面；
2. 生成密钥对。输入命令：ssh-keygen -t rsa -C回车三次。找到里面的.ssh路径，打开.ssh文件，找到id_rsa.pub并用记事本打开，这就是我们需要的密钥。
3. 拿到密钥后，回到Git Bash命令界面，输入命令：ssh -T git@github.com检测是否成功绑定Github。
a. 在这里我出现了Connection reset by 192.30.253.112 port 22
b. 在防火墙高级安全中新建入站规则，选择端口22，允许连接，输入名称和描述 。
c. 然后回到Git Bash上重新输入ssh -T git@github.com便可成功检测到Github绑定成功。
4. 然后在Git Bash上进入blog目录下输入hexo d即可成功部署到远端。

主题

在克隆主题时出现RPC failed，无法成功克隆，因为postBuffer的默认值太小，需要调整它的大小。
解决方法：在Git Bash上输入git config –list命令，重新配置postBuffer的值。这里我改的值是：http.postbuffer=524288000。

Over!
